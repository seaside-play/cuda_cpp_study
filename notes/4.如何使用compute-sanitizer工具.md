compute-sanitizer 完整使用教程（CUDA 内存 / 越界报错排查，替代旧 cuda-memcheck）
一、前置要求
CUDA Toolkit ≥ 11.3 才自带 compute-sanitizer
编译程序必须加 -lineinfo，否则只会报地址，不显示源码行号
不建议加 -G（关闭设备优化会大幅降低运行速度，仅本地临时调试）
二、基础编译命令（关键）
bash
运行
# add1.cu 你的cuda源码
nvcc -lineinfo add1.cu -o add_demo
-lineinfo：保留源码行号，sanitizer 能定位到具体代码行
不要去掉，否则报错只会打印 SASS 地址，看不懂
三、最简运行（内存越界检测，最常用）
1. 基础内存检查（memcheck，排查 719 launch failure）
bash
运行
compute-sanitizer --tool memcheck ./add_demo
能检测：
全局 / 共享 / 局部内存越界读写
空指针访问
未初始化内存读取
数组索引溢出（你遇到的 unspecified launch failure 90% 靠这个定位）
2. 并发竞争检测（racecheck）
多线程同时写同一块内存、无同步导致数值错乱
bash
运行
compute-sanitizer --tool racecheck ./add_demo
3. 未初始化内存专项检测（initcheck）
读取未赋值的设备内存，出现随机 NaN、异常数值
bash
运行
compute-sanitizer --tool initcheck ./add_demo
四、高频实用参数（工业调试必用）
1. 输出详细堆栈，打印线程 / 块 ID
bash
运行
compute-sanitizer --tool memcheck --print-lines --show-backtrace ./add_demo
--print-lines：输出源码行号
--show-backtrace：完整调用栈，显示出错 threadIdx/blockIdx
2. 保存报告到文件，方便后期分析
bash
运行
compute-sanitizer --tool memcheck --log-file sanitizer_log.txt ./add_demo
3. 忽略无关报错，只看前 N 条（报错 64 条时减少刷屏）
bash
运行
compute-sanitizer --tool memcheck --max-errors 10 ./add_demo
4. 开启内核同步详细日志
bash
运行
compute-sanitizer --tool memcheck --check-stack ./add_demo
五、完整示例（解决你之前 719 错误场景）
编译带行号
bash
运行
nvcc -lineinfo add1.cu -o add
执行 sanitizer 内存检测
bash
运行
compute-sanitizer --tool memcheck --print-lines ./add
输出解读示例：
plaintext
========= Invalid __global__ write of size 4
=========     at add1.cu:42
=========     Thread block (1,0,0), thread (32,0,0)
=========     Address 0x7f1234000 out of bounds
含义：
第 42 行，线程 threadIdx.x=32 发生全局内存写越界
直接定位数组长度不足、索引计算错误
六、常见报错对应工具
unspecified launch failure 719、程序直接崩溃 → memcheck
计算结果随机错误、数值忽大忽小 → racecheck
输出大量 NaN、随机乱数 → initcheck
七、注意事项（踩坑点）
性能损耗极大
sanitizer 运行速度比正常程序慢几十～上百倍，绝对不要在线上 / 集群长期跑，仅线下复现 bug 使用。
动态共享内存溢出也能捕获
静态 / 动态__shared__分配超出硬件上限，会明确提示 smem overflow。
__syncthreads()分支死锁无法被 sanitizer 捕获
同步死锁需要人工检查 if 分支内同步代码。
多卡程序需要指定 GPU
bash
运行
CUDA_VISIBLE_DEVICES=0 compute-sanitizer --tool memcheck ./add
容器 / 无图形服务器完全支持，纯命令行，无需 Nsight
八、和旧 cuda-memcheck 对比
cuda-memcheck：CUDA11.3 前工具，已官方淘汰
compute-sanitizer：全新重构，报错更精准、支持更多检测类型、堆栈信息更完整
新项目统一用 compute-sanitizer。
九、一键调试脚本（直接复制使用）
bash
运行
# 编译+内存检测一键执行
nvcc -lineinfo add1.cu -o app
compute-sanitizer --tool memcheck --print-lines --max-errors 20 ./app
