CUDA error 719 unspecified launch failure 完整排查方案
一、错误含义
719 = cudaErrorUnspecifiedLaunchFailure
内核启动后在 GPU 执行时发生致命异常，驱动无法给出精确细分报错，统一归为未指定启动失败，底层常见诱因：越界访存、共享内存溢出、warp 非法操作、硬件报错、寄存器爆栈、空指针、同步依赖死锁等。
ERROR SUMMARY:64 errors = 一共 64 次 GPU 异常触发。
二、最快定位工具（工业标准）
1. compute-sanitizer（优先，CUDA11.3+）
bash
运行
# 编译必须加 -lineinfo
nvcc -lineinfo add1.cu -o add1
# 内存/越界检测
compute-sanitizer ./add1
会精准输出：出错线程 block/thread、源码行、读写地址、越界类型（全局 / 共享 / 局部内存）。
旧版本替代：cuda-memcheck ./add1
2. 捕获精确异步错误（不加工具时本地调试）
CUDA kernel 启动是异步，报错不会立刻返回，必须在 launch 后同步并捕获：
cpp
运行
// 核函数调用
add_kernel<<<grid, block>>>(...);
// 同步捕获GPU异常
cudaError_t err = cudaGetLastError();
if(err != cudaSuccess){
    printf("launch error: %s\n", cudaGetErrorString(err));
}
cudaDeviceSynchronize();
err = cudaGetLastError();
if(err != cudaSuccess){
    printf("sync error line44: %s\n", cudaGetErrorString(err));
}
你报错行是 44，大概率是kernel<<<>>>这一行。
三、高频触发原因 & 修复
1. 全局内存越界读写（最常见）
数组分配大小不足，thread 索引超出数组长度；
传入主机指针到核函数（未 cudaMalloc/cudaMemcpy）；
设备指针为 nullptr。
修复：核对数组长度、threadIdx/blockIdx 计算逻辑，打印 tid 核对范围。
2. 共享内存 shared memory 溢出
cpp
运行
// 动态共享内存
__global__ void kernel(float* out){
    __shared__ float sh[1024]; // 静态超硬件限制
    // 或 <<<>>> 传入的动态smem字节数超出SM上限
}
单 SM 共享内存上限：
sm_61:48KB；sm_86:163KB；sm_89:163KB
超出直接触发 launch failure。
3. 寄存器占用过高，occupancy 为 0
cuobjdump 查看 kernel 寄存器：
bash
运行
cuobjdump -sass add1.cubin
Registers/thread数值过大，超出硬件寄存器池，内核无法调度。
缓解：__launch_bounds__(128)限制单块线程数，减少寄存器占用。
4. 同步死锁（__syncthreads）
if 分支内写__syncthreads()，同一 warp 部分线程跳过同步；
block 内线程未全部到达同步点，GPU 卡死抛 719。
示例错误：
cpp
运行
if(threadIdx.x > 16){
    __syncthreads(); // 一半线程不执行，死锁崩溃
}
5. 数值非法运算（部分架构会触发）
除零、log (负数)、sqrt (-1)、无穷 NaN 参与原子操作，部分 GPU 架构抛出 launch 异常。
6. 网格 / 块参数非法
blockDim.x > 1024；gridDim 超大导致资源耗尽；动态共享内存传入负数。
四、分步排查流程（按顺序操作）
编译加 -lineinfo，用compute-sanitizer运行，看精确越界堆栈；
注释核内一半逻辑二分定位出错代码块；
打印 thread 索引，确认不会超过数组边界；
检查所有__syncthreads()是否在统一分支；
查看共享内存静态 / 动态分配是否超标；
cuobjdump 核对寄存器占用；
缩小 grid/block 尺寸测试，判断是否资源过载；
关闭所有设备端 printf（大量 printf 会驱动缓冲区溢出崩溃）。
五、补充：集群 / 服务器特殊情况
多卡程序未指定设备、GPU 被其他进程占用、ECC 内存硬件报错、驱动版本与 CUDA 不匹配，也会批量爆出数十条 719 错误。
增加设备选择代码：
cpp
运行
cudaSetDevice(0);
