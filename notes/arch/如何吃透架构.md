一、先吃透顶层芯片组织（Ada vs Blackwell 整体结构）
1. 统一层级模型（两代通用）

2. Ada（RTX40 sm_89）关键硬件参数
完整 AD102（RTX4090）：
12 GPC，72 TPC，144 个 SM
单 SM：128 FP32 CUDA Core、4 个第三代 RT Core、8 个第四代 Tensor Core
SM 片上存储：128KB 可分配 L1/Shared Memory（96KB SMEM+32KB L1 / 64+64 / 32+96 三档可调）
L2 缓存：72MB（4090），带宽 1TB/s+
调度：双 Warp 调度器，单周期可同时发射浮点 + 整数指令（双发射 ILP）
3. Blackwell（sm_100，RTX50/GB100）核心差异
完整 GB202（RTX5090/RTX PRO6000）：
12 GPC，96 TPC，192 个 SM（比 Ada 旗舰多 33% SM）
单 SM FP32 吞吐量翻倍，整数运算能力大幅提升（神经着色、地址生成加速）
第五代 Tensor Core：新增 FP4、第二代 FP8 Transformer 加速，引入TMEM 张量专用内存（Ada 无）
第四代 RT Core，硬件 BVH 优化，光流单元升级
消费级 GB20x：单 SM 仍 128KB L1/SMEM；数据中心 GB100 SMEM 提升至 228KB
L2 缓存最高 128MB，搭配 GDDR7 显存，带宽翻倍
TPC 双 SM 支持协作 CTA（2CTA 联合 MMA 矩阵运算，Ada 不支持）
二、深度拆解 SM 微架构（重中之重，决定所有 CUDA 优化）

SM内部完整结构图
1. SM 固定硬件组成（Ada/Blackwell 通用基础）
Warp 调度器：最小调度单位 32 线程 = 1 个 warp；sm_89/sm_100 单 SM 最大并发 64 个 warp
4 组独立寄存器堆：总 65536 个 32bit 通用寄存器；每个线程寄存器用量直接限制 Occupancy 占用率
运算单元分区：
FP32/INT32 CUDA Core：普通浮点、整数计算
SFU 特殊函数单元：sin/cos/log 等
Tensor Core：矩阵乘 MMA（AI / 卷积核心）
RT Core：光线相交测试
L0 指令缓存、L1 数据缓存 + Shared Memory 共享内存
LD/ST 加载存储单元，全局内存事务通路
2. Ada SM（sm_89）独有特性
双发射流水线：同一时钟周期同时执行一条 FP32 和一条 INT32 指令，消除整数阻塞
L1/SMEM 三档动态划分，适合图形 / 计算切换场景
第四代 Tensor Core 支持 FP16/BF16/FP8，无 FP4，无 TMEM
Shared Memory Bank 冲突基础 Swizzling 置换算法
3. Blackwell SM（sm_100）革命性升级
TMEM 张量内存：MMA 累加器不再占用通用寄存器，大幅释放寄存器资源，大模型算子占用率暴涨
增强 Shared Memory Swizzling：多阶地址异或，大幅降低不规则访存 Bank 冲突
TPC 级协作 CTA：同 TPC 两个 SM 共享张量操作数，减少 SMEM 带宽消耗
FP4 低精度硬件原生支持，推理吞吐量翻倍
数据中心版 SMEM 扩容至 228KB，超大 Tile 矩阵计算无需频繁读写全局显存
4. 必须吃透的核心硬件约束（写 CUDA 的底层依据）
Occupancy 占用率三大约束（木桶效应）
寄存器数量：线程用寄存器越多，能并行的 warp 越少
Shared Memory 大小：Block 占用共享内存越大，单 SM 容纳 Block 越少
最大 Block 上限：sm_89/sm_100 单 SM 最多 32 个 Block
Warp 分支发散：SIMT 架构硬件天然短板，两代均靠分支栈优化，Blackwell 分支处理效率小幅提升
共享内存 Bank：32 个 Bank，连续 32 线程对齐访问无冲突；Blackwell 置换算法缓解不规则步长冲突
三、存储层级与访存硬件（解决你之前 coalesced 合并访存疑问）
1. 从快到慢完整层级（硬件延迟、带宽天差地别）
寄存器（Register）：单线程私有，1 周期访问，带宽最高，容量极小
Shared Memory（SMEM）：Block 内线程共享，单 SM 片上，几周期延迟，软件手动管理
L1 Cache：SM 私有，硬件自动缓存全局内存
L2 Cache：GPU 全局统一缓存，所有 SM 共享
Global 显存（GDDR6X/GDDR7/HBM）：最慢，数百周期延迟，带宽瓶颈
2. 合并访存 Coalesced 硬件底层原理
GPU LD/ST 单元会把一个 warp（32 线程）的全局内存地址打包为最少内存事务：
连续、64/128/256 字节对齐地址 → 1 次事务 = Fully Coalesced，带宽拉满
散乱跳跃地址 → 拆分成数十次独立事务，带宽暴跌
Ada 与 Blackwell 底层事务打包逻辑一致，但 Blackwell L2 更大、GDDR7 带宽更高，访存惩罚更小。
两代存储关键差异
Ada：GDDR6X，最高 72MB L2；SMEM 最大 96KB/Block
Blackwell 消费级：GDDR7，最高 128MB L2；新增 TMEM 张量专用高速存储；数据中心版 SMEM 228KB

阶段 2：工具实测硬件信息（直观验证硬件参数）
deviceQuery.cu（CUDA 自带示例）
输出 SM 总数、算力、单 SM 最大共享内存、寄存器、最大 warp/Block、L2 大小，快速对比你的 RTX40 和 RTX50 硬件规格。
bandwidthTest：实测全局显存带宽，对比 Ada/Blackwell 理论带宽差距
occupancy示例：计算不同寄存器 / SMEM 占用下的 SM 理论占用率，理解硬件资源约束
阶段 3：Nsight Compute 硬件级性能剖析（核心实操）
学习重点：
SM 指标：Active Warps、Warp Stall 原因（寄存器等待、访存延迟、分支发散）
存储指标：L1/L2 缓存命中率、Global Load/Store 事务数量、是否 Coalesced
资源占用：每线程寄存器用量、Shared Memory Bank 冲突次数
Tensor Core 利用率：Ada 无 TMEM vs Blackwell TMEM 带来的吞吐提升
实操实验：
写向量加法 kernel，对比合并访存 vs 非合并访存的事务数、耗时差距
矩阵乘法调优：调整 Tile 大小，观察 SMEM 占用、Occupancy、Tensor Core 利用率变化
同一代码分别编译 -arch=sm_89 / -arch=sm_100，在对应显卡上跑，对比硬件流水线差异
阶段 4：微基准测试，量化两代架构硬件延迟
自己写极简 kernel 测量：
寄存器访问延迟、Shared Memory Bank 冲突延迟
Global 内存未命中 L1/L2 的访存延迟
Tensor Core MMA 单指令吞吐（Ada FP8 vs Blackwell FP4）
Warp 分支发散带来的性能损耗
通过量化数据，把白皮书文字转化为可感知的性能差异。
阶段 5：编译底层理解（PTX/SASS，打通软硬件）
nvcc 生成 PTX 中间汇编：nvcc -ptx kernel.cu，看懂 warp、内存、MMA 指令
cuobjdump 看 SASS 硬件指令：cuobjdump -sass a.out
sm_89 SASS：无 TMEM 相关指令
sm_100 SASS：新增 TMEM 加载存储、2CTA 协作 MMA 指令
理解编译指令如何映射到 SM 内部硬件执行单元，彻底打通软件 - 硬件链路。
五、Ada (sm_89) vs Blackwell (sm_100) 核心差异速记（面试 / 优化高频）
表格
维度	Ada Lovelace sm_89(RTX40)	Blackwell sm_100(RTX50/GB100)
单卡最大 SM	144（AD102）	192（GB202）
Tensor Core	四代，FP16/BF16/FP8，无 TMEM	五代，FP4 / 二代 FP8，TMEM 张量内存
SM 共享内存	最高 96KB/Block，128KB 总 L1+SMEM	消费级同；数据中心 228KB/Block
协作计算	无跨 SM 协作 CTA	TPC 内双 SM 支持 2CTA 联合 MMA
显存	GDDR6X	GDDR7，PAM3 信号，带宽翻倍
L2 缓存	最大 72MB	最大 128MB
整数吞吐	基础单发射	每时钟整数吞吐量翻倍
Shared Memory 冲突优化	基础置换算法	多阶地址 Swizzling，大幅降低冲突
