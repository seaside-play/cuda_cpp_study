cubin 程序完整调试方案
前置基础
cubin 是 GPU 二进制机器码文件（nvcc -code=sm_xx 输出），不含主机代码，仅设备微码，无法直接运行，只能被 CUDA 驱动加载执行；调试分 4 类场景：源码级调试、cubin 汇编调试、运行时报错定位、离线静态分析。
一、编译开启调试符号（必须第一步）
1. 编译带调试信息的 cubin
关键参数：
-g：设备代码调试符号
-G：完全设备调试（关闭设备优化，变量、行号完整映射，调试首选）
-lineinfo：仅保留行号，不关闭优化（性能调试用）
-cubin：直接输出 cubin 文件
示例命令：
bash
运行
# 生成带调试符号 cubin，无优化，适合gdb/nsight调试
nvcc -arch=sm_86 -G -g -cubin test.cu -o test_sm86.cubin

# 保留优化+行号（性能分析场景）
nvcc -arch=sm_86 -lineinfo -cubin test.cu -o test_opt_sm86.cubin
参数区别
-G -g：禁用设备端所有优化，变量不被寄存器消除，支持完整断点、局部变量查看；
-lineinfo：保留优化，仅记录源码行号，无法打印局部变量，仅用于性能采样。
二、方案 1：Nsight Compute / Nsight Visual Studio Edition（图形化，最推荐）
Nsight 原生支持加载 cubin + 对应源码做设备调试。
流程
编译 cubin 时带上 -G -g；
编写极简 host 加载程序，用 cuModuleLoad 加载 cubin、cuLaunchKernel 启动核函数；
Nsight Attach 进程，或直接 Launch 主机程序；
功能：
在 CUDA C++ 源码打断点，命中后自动映射 cubin SASS 汇编；
查看寄存器、共享内存、全局内存、局部变量；
单步执行 GPU 线程，切换 warp/block/thread；
查看 cubin 内部 SASS 微码，对照源码逐指令调试。
查看 cubin 汇编（SASS）
Nsight 中右键核函数 → View SASS，直接展示 cubin 解码后的 GPU 机器指令，可和源码对照。
三、方案 2：cuda-gdb 命令行源码调试
cuda-gdb 是 CUDA 官方命令行调试器，支持调试加载 cubin 的主机程序。
编写 host 代码手动加载 cubin 并启动 kernel：
cpp
运行
#include <cuda_runtime.h>
int main(){
    CUmodule mod;
    CUfunction ker;
    cuModuleLoad(&mod, "test_sm86.cubin");
    cuModuleGetFunction(&ker, mod, "_Z6kerneli"); // mangled核函数名
    // 配置参数、launch kernel
    // ...
    return 0;
}
编译主机代码：g++ host.cpp -lcuda -o host
启动调试：
bash
运行
cuda-gdb ./host
(gdb) run
# GPU断点
(gdb) break kernel.cu:20
(gdb) c
# 切换GPU线程
(gdb) cuda thread 0
# 打印设备变量
(gdb) print tid
# 查看SASS汇编
(gdb) disas $pc
限制：cuda-gdb 不能直接打开 cubin，必须依托 host 程序动态加载执行。
四、方案 3：离线解析 cubin（不运行 GPU，静态分析）
1. nvdisasm：官方 cubin 反汇编工具（核心工具）
nvdisasm 可以把 cubin 解码为可读 SASS GPU 汇编，带调试符号时会附带源码行注释。
bash
运行
# 基础反汇编输出SASS
nvdisasm test_sm86.cubin > kernel_sass.txt

# 带源码行号注释（编译加了-G才生效）
nvdisasm -g test_sm86.cubin > kernel_debug_sass.txt

# 只反汇编指定核函数
nvdisasm -func kernel test_sm86.cubin
输出内容：
寄存器使用、共享内存大小、分支、内存读写指令；
带 -G 时每条 SASS 指令上方标注对应 CUDA C++ 源码行；
用于：定位越界访问、分支发散、重复内存读写、寄存器溢出。
2. cuobjdump：查看 cubin 元信息
查看 cubin 内符号、常量、全局内存、内核参数、寄存器占用：
bash
运行
cuobjdump test_sm86.cubin
# 提取所有设备符号
cuobjdump -symbols test_sm86.cubin
# 查看kernel占用寄存器/共享内存
cuobjdump -ptx test_sm86.cubin
五、方案 4：运行时错误调试（cubin 执行崩溃定位）
1. 开启 CUDA 错误检查
host 调用 cuModuleLoad/cuLaunchKernel 后立即检查返回值：
cpp
运行
CUresult res = cuModuleLoad(&mod, "test.cubin");
if(res != CUDA_SUCCESS){
    const char* err;
    cuGetErrorString(res, &err);
    printf("load cubin fail: %s\n", err);
}
常见 cubin 加载错误：
CUDA_ERROR_INVALID_IMAGE：cubin sm 和当前 GPU 算力不匹配；
CUDA_ERROR_FILE_NOT_FOUND：文件路径错误；
CUDA_ERROR_COMPAT_NOT_SUPPORTED_ON_DEVICE：CUDA 版本不兼容。
2. 设备核函数崩溃（空指针、越界）
主机端每次 launch 后调用同步：cudaDeviceSynchronize()；
捕获同步错误，配合 cuda-memcheck 内存检查：
bash
运行
cuda-memcheck ./host
cuda-memcheck 会追踪 cubin 内核中全局 / 共享内存越界、空指针、race condition，输出出错线程、源码行、SASS 指令地址。
六、多场景调试使用建议
源码断点、单步调试、看变量 → Nsight Compute /cuda-gdb，编译加 -G -g；
不运行 GPU，静态看 GPU 汇编、分析指令 → nvdisasm；
内存越界、野指针、并发冲突 → cuda-memcheck；
仅查看 cubin 元数据、寄存器占用、内核符号 → cuobjdump；
性能瓶颈分析（不关闭优化）：编译加 -lineinfo，Nsight Compute Profiler。
七、常见坑
只用 -g 不加 -G：设备优化开启，局部变量被优化，调试看不到值；
cubin sm_xx 高于本机 GPU 算力：cuModuleLoad 直接报错；
无调试符号的 cubin：nvdisasm 不会附带源码注释，只能裸看汇编；
不能直接执行 cubin：必须写主机代码调用 CUDA Driver API 加载运行；
CUDA13 不再支持 sm_6x/sm_70 cubin，旧显卡无法加载高版本编译的 cubin。
